## ğŸ’¡ Pro Tips

### Commit Often
Small, frequent commits are better than large ones:
```powershell
# Good
git commit -m "feat: Add RSI calculator"
git commit -m "test: Add RSI unit tests"
git commit -m "docs: Document RSI feature"

# Not ideal
git commit -m "Added everything"
```

### Write Clear Messages
Future you will appreciate it:
```powershell
# Clear
git commit -m "fix: Handle division by zero in ROE calculation"

# Unclear
git commit -m "fixed bug"
```

### Use Branches for Experiments
```powershell
# Create feature branch
git checkout -b feature/new-indicator

# Work on it
git add .
git commit -m "feat: Add new indicator"

# Merge when ready
git checkout main
git merge feature/new-indicator
```
